cardinalities

When openEHR 0..* and FHIR 0..* the corresponding iteration is to be done. e.g.

  - name: "diagnose"
    with:
      fhir: "$resourcePath.diagnosis"
      openEHR: "$reference"
      type: "NONE"

These can be a list of diagnoses therefore iterate this mapping for each of the elements ## context mappings each context mapping has its own imports, so its not mixed when cross referencing

writing into 0..* fields

E.g. mappings from a 0..* dosage that contains a route into the openEHR route which is 0..n. Field is added for each cluster again and again. if its 0..* append, if its 0..1 overwrite. Mapping to List in openEHR should always append to list e.g. /links.
